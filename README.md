# MicroGit

保存と同時に裏で自動コミットを行う、タイムトラベル・超高頻度履歴管理ツールです。

`.microgit_shadow` は大元リポジトリのコミットに含める前提です。対象ブランチ上の通常コミットを復元すると、その時点のマイクロ履歴も一緒に戻り、マイクロブランチ単位でさかのぼれます。GitHub などリモートへシャドウ履歴を push することはありません。

## 主な機能

- **オン / オフ切り替え**: ステータスバーまたはコマンドパレットから有効化・無効化できます。有効化したブランチが「対象ブランチ」になります。
- **対象ブランチ限定のシャドウ**: 対象ブランチ以外へ切り替えると、作業ツリー上の `.microgit_shadow` / `.microgit_logs` を自動削除し、他ブランチのコミットへ混入しないようにします。
- **全自動シャドウコミット**: 有効かつ対象ブランチ上でファイルを保存すると、`.microgit_shadow` に自動でコミットが残ります。
- **直感的なタイムトラベル**: 過去の保存ポイントへ、ボタン一つでワークスペースのファイルを復元できます。
- **タイムライングラフ**: 専用の Webview ビューアで、分岐したマイクロ履歴を視覚的に確認できます。

## 使い方

1. **有効化**: ステータスバーの `MicroGit: OFF` をクリックするか、コマンドパレットで `MicroGit: Enable` を実行します。このときチェックアウト中のブランチが対象ブランチになります。
2. **履歴の自動記録**: 対象ブランチ上でファイルを保存（`Ctrl + S` / `Cmd + S`）すると履歴が蓄積されます。必要に応じて `.microgit_shadow` を大元リポジトリへ通常コミットしてください。
3. **グラフの表示**: コマンドパレットで `MicroGit: Show Graph` を実行します。
4. **他ブランチへ移動**: 対象外ブランチではシャドウ領域が自動削除されます。対象ブランチ（とそのコミット）へ戻ると、コミット済みのマイクロ履歴が Git によって復元されます。
